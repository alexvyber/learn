Basic SQL Questions

  1. What is SQL?

    • SQL (Structured Query Language) is a standard programming language used for managing and manipulating relational databases.

  2. What are the different types of SQL commands?

    • DDL (Data Definition Language), DML (Data Manipulation Language), DCL (Data Control Language), TCL (Transaction Control Language), and DQL (Data Query Language).

  3. What is a primary key?

    • A primary key is a unique identifier for a record in a table. It cannot contain NULL values and must contain unique values.

  4. What is a foreign key?

    • A foreign key is a column or a set of columns in one table that uniquely identifies rows in another table.

  5. What is a JOIN in SQL?

    • A JOIN is used to combine rows from two or more tables based on a related column between them.

  6. What are the different types of JOINS in SQL?

    • INNER JOIN, LEFT JOIN, RIGHT JOIN, FULL OUTER JOIN, CROSS JOIN, and SELF JOIN.

  7. What is the difference between INNER JOIN and OUTER JOIN?

    • INNER JOIN returns records that have matching values in both tables, while OUTER JOIN returns all records from one table and matched records from the other table.

  8. What is a NULL value in SQL?

    • NULL represents a missing or undefined value in a database. It is not the same as zero or an empty string.

  9. What is the difference between WHERE and HAVING clauses?

    • WHERE is used to filter records before any groupings are made, while HAVING is used to filter groups after the GROUP BY clause is applied.

  10. What is the GROUP BY clause?

    • GROUP BY is used to arrange identical data into groups with the help of aggregate functions like COUNT, SUM, AVG, etc.

Intermediate SQL Questions

  1. What is an index in SQL?

    • An index is a database object that improves the speed of data retrieval operations on a table at the cost of additional storage space.

  2. What is the difference between a clustered and a non-clustered index?

    • A clustered index sorts and stores the data rows of the table based on the index key, while a non-clustered index creates a separate structure that points back to the original table rows.

  3. What is a subquery in SQL?

    • A subquery is a query nested within another SQL query. It can be used in SELECT, INSERT, UPDATE, or DELETE statements.

  4. What is a correlated subquery?

    • A correlated subquery is a subquery that references columns from the outer query and is executed for each row of the outer query.

  5. What is normalization?

    • Normalization is the process of organizing data to minimize redundancy and dependency by dividing large tables into smaller, related tables.

  6. Explain the different normal forms.

    • 1NF: Ensures that each column contains atomic values and each column contains unique values.

    • 2NF: Satisfies 1NF and ensures that all non-key attributes are fully functional dependent on the primary key.

    • 3NF: Satisfies 2NF and ensures that no transitive dependency exists.

    • BCNF: A stronger version of 3NF where every determinant is a candidate key.

  7. What is denormalization?

    • Denormalization is the process of combining normalized tables into larger tables to improve query performance at the cost of redundancy.

  8. What is a view in SQL?

    • A view is a virtual table that consists of a subset of data from one or more tables. Views can simplify complex queries and enhance security by restricting access to specific data.

  9. What is a stored procedure?

    • A stored procedure is a precompiled collection of one or more SQL statements that can be executed as a single unit, often used for reusability and performance optimization.

  10. What is a trigger in SQL?

    • A trigger is a set of SQL statements that automatically execute in response to certain events on a table, such as INSERT, UPDATE, or DELETE.


Advanced SQL Questions

  1. What is ACID in SQL?

    • ACID stands for Atomicity, Consistency, Isolation, Durability. These are properties that ensure reliable processing of database transactions.

  2. What is a transaction in SQL?

    • A transaction is a sequence of one or more SQL operations that are treated as a single unit of work, ensuring that either all operations succeed or none at all.

  3. What are the types of transactions in SQL?

    • Explicit, Implicit, Autocommit, and Nested transactions.

  4. What is SQL injection?

    • SQL injection is a security vulnerability that allows attackers to execute arbitrary SQL code on a database by inserting malicious queries into input fields.

  5. How can you prevent SQL injection?

    • Use parameterized queries, prepared statements, and input validation to prevent SQL injection.

  6. What is a CTE (Common Table Expression)?

    • A CTE is a temporary result set that you can reference within a SELECT, INSERT, UPDATE, or DELETE statement. It is defined using the WITH clause.

  7. What is the difference between RANK() and DENSE_RANK() in SQL?

    • RANK() assigns a rank to each row with gaps in ranking, while DENSE_RANK() assigns a rank without gaps.

  8. What is the difference between DELETE and TRUNCATE commands?

    • DELETE removes rows based on a condition and can be rolled back. TRUNCATE removes all rows without conditions and cannot be rolled back in most databases.

  9. What is the difference between UNION and UNION ALL?

    • UNION combines the result set of two queries and removes duplicates, while UNION ALL combines the result sets and includes duplicates.

  10. What is a materialized view?

    • A materialized view is a database object that contains the results of a query and is stored physically. Unlike regular views, it does not require recalculating the data every time it is queried.

  11. What is indexing in SQL?

    • Indexing is a technique used to improve the speed of data retrieval operations on a database table at the cost of additional storage space.

  12. What is a composite key?

    • A composite key is a primary key made up of two or more columns, which together uniquely identify a record.

  13. What are the aggregate functions in SQL?

    • COUNT(), SUM(), AVG(), MAX(), MIN() are some common aggregate functions used to perform calculations on a set of values.

  14. What is the difference between COUNT() and COUNT(1)?

    • COUNT() counts the number of rows with non-NULL values in a column, while COUNT(1) counts all rows, including those with NULL values.

  15. What is the purpose of the DISTINCT keyword?

    • DISTINCT is used to remove duplicate records from the result set of a SELECT query.

  16. What is a recursive query?

    • A recursive query is a query that refers to itself, often used to query hierarchical or tree-structured data.

  17. What is the difference between a scalar function and a table-valued function?

    • A scalar function returns a single value, whereas a table-valued function returns a table.

  18. What is the difference between a CROSS JOIN and an INNER JOIN?

    • CROSS JOIN returns the Cartesian product of two tables (all possible combinations), while INNER JOIN returns rows with matching values in both tables.

  19. What is a surrogate key?

    • A surrogate key is an artificial key (often an auto-incrementing number) used as a unique identifier for records in a table, not derived from the application data.

  20. What is a pivot table in SQL?

    • A pivot table is a data summarization tool used to transpose and aggregate data from rows into columns.


Optimization and Performance Tuning

  1. What are the common performance issues in SQL?

    • Slow queries, inefficient joins, lack of indexing, excessive data retrieval, and improper use of subqueries.

  2. How can you optimize a SQL query?

    • Use proper indexing, avoid unnecessary columns in SELECT, use WHERE clauses effectively, and avoid complex joins or subqueries when possible.

  3. What is the EXPLAIN command in SQL?

    • The EXPLAIN command is used to analyze and display the execution plan of a query, showing how the database will execute the query.

  4. What is a covering index?

    • A covering index is an index that contains all the columns required by a query, eliminating the need to access the table data.

  5. What is a query execution plan?

    • A query execution plan is a detailed roadmap provided by the database engine on how it will execute a query, including operations like table scans, index usage, and joins.

  6. What is the difference between a full table scan and an index scan?

    • A full table scan reads all rows in a table, while an index scan reads only the indexed columns, which is generally faster.

  7. What is a partitioned table in SQL?

    • A partitioned table is a large table divided into smaller, more manageable pieces, called partitions, often used to improve query performance and manageability.

  8. What is the impact of using the LIKE operator on performance?

    • The LIKE operator can be slow, especially when using leading wildcards (e.g., %search_term), as it may prevent the use of indexes.

  9. What is query optimization?

    • Query optimization is the process of modifying a SQL query to make it run faster by improving its execution plan.

  10. What is the role of statistics in SQL performance?

    • Statistics provide the database engine with information about data distribution, which helps the query optimizer choose the most efficient execution plan.


Transactions and Concurrency Control

  1. What is the difference between COMMIT and ROLLBACK?

    • COMMIT saves all changes made during a transaction, while ROLLBACK undoes all changes made during a transaction.

  2. What is a savepoint in SQL?

    • A savepoint allows you to set a point within a transaction to which you can later roll back without affecting the entire transaction.

  3. What are isolation levels in SQL?

    • Isolation levels define the degree to which a transaction must be isolated from other transactions, including Read Uncommitted, Read Committed, Repeatable Read, and Serializable.

  4. What is a deadlock in SQL?

    • A deadlock occurs when two or more transactions hold locks on resources that the other transactions need, causing them to wait indefinitely.

  5. How can you prevent deadlocks in SQL?

    • By ensuring consistent ordering of resource access, minimizing transaction times, and using proper isolation levels.

  6. What is the difference between optimistic and pessimistic concurrency control?

    • Optimistic concurrency control assumes that multiple transactions can complete without affecting each other, while pessimistic concurrency control locks resources to prevent conflicts.

  7. What is a lock in SQL?

    • A lock is a mechanism to control access to a database resource during concurrent transactions to ensure data integrity.

  8. What is a phantom read?

    • A phantom read occurs when a transaction re-reads data and finds rows that were not visible during the first read due to another transaction inserting or deleting rows.

  9. What is two-phase commit in SQL?

    • Two-phase commit is a protocol used to ensure all participants in a distributed transaction either commit or rollback changes in a coordinated manner.

  10. What is a transaction log in SQL?

    • A transaction log is a file that records all transactions and changes made to the database, used for recovery and rollback purposes.


Miscellaneous Questions

  1. What is SQL injection and how can it be prevented?

    • SQL injection is a code injection technique that exploits vulnerabilities in SQL queries to execute arbitrary SQL code. It can be prevented using parameterized queries and input validation.

  2. What are user-defined functions in SQL?

    • User-defined functions (UDFs) are custom functions written by users to perform complex calculations and can return a scalar value or a table.

  3. What is the difference between OLTP and OLAP?

    • OLTP (Online Transaction Processing) is used for transactional databases, focusing on speed and data integrity. OLAP (Online Analytical Processing) is used for complex queries and data analysis.

  4. What is a cursor in SQL?

    • A cursor is a database object used to retrieve, manipulate, and traverse through a result set row by row.

  5. What is the difference between an implicit and an explicit cursor?

    • Implicit cursors are automatically created by the database when a query is executed, while explicit cursors are defined and controlled by the user.

  6. What is the purpose of the CASE statement in SQL?

    • The CASE statement allows you to perform conditional logic in SQL queries, similar to an IF-THEN-ELSE statement.

  7. What is the difference between a sequence and an auto-increment in SQL?

    • A sequence is a database object that generates unique numbers in a defined order, while auto-increment automatically generates unique values for a column.

  8. What is the purpose of the MERGE statement in SQL?

    • The MERGE statement allows you to perform INSERT, UPDATE, or DELETE operations on a table based on the results of a join with another table.

  9. What is the difference between DELETE and DROP commands?

    • DELETE removes rows from a table based on a condition, while DROP removes the entire table or database.

  10. What is referential integrity?

    • Referential integrity ensures that relationships between tables remain consistent, specifically that foreign keys correctly reference primary keys in related tables.

  11. What is the difference between CHAR and VARCHAR data types?

    • CHAR is a fixed-length character data type, while VARCHAR is a variable-length character data type.

  12. What is the difference between INNER JOIN and OUTER JOIN?

    • INNER JOIN returns rows with matching values in both tables, while OUTER JOIN returns all rows from one table and the matched rows from the other table.

  13. What is the difference between SQL and NoSQL?

    • SQL databases are relational and use structured query language, while NoSQL databases are non-relational and can store unstructured data.

  14. What is database normalization?

    • Database normalization is the process of organizing data into tables to minimize redundancy and improve data integrity.

  15. What is a foreign key constraint?

    • A foreign key constraint enforces a link between the data in two tables, ensuring that data in the foreign key column matches data in the referenced table's primary key column.

  16. What is a NULL value in SQL?

    • A NULL value represents a missing or undefined value in a database.

  17. What is a unique constraint in SQL?

    • A unique constraint ensures that all values in a column or group of columns are distinct from each other.

  18. What is the difference between COUNT(*) and COUNT(column_name)?

    • COUNT(*) counts all rows in a table, including those with NULL values, while COUNT(column_name) counts only non-NULL values in a specific column.

  19. What is a surrogate key?

    • A surrogate key is a unique identifier for a record in a table, often generated automatically by the database.

  20. What is a composite key?

    • A composite key is a primary key made up of two or more columns.


Advanced Optimization and Best Practices

  1. How can you improve query performance in SQL?

    • By using indexing, optimizing query structure, minimizing subqueries, and avoiding unnecessary columns in SELECT statements.

  2. What is the purpose of the EXPLAIN command in SQL?

    • The EXPLAIN command provides details about how a SQL query will be executed, including the use of indexes and join methods.

  3. What is the difference between a covering index and a composite index?

    • A covering index includes all the columns needed to satisfy a query, while a composite index is an index on multiple columns.

  4. What is the purpose of partitioning in SQL?

    • Partitioning divides a table into smaller, more manageable pieces, improving query performance and making large datasets easier to manage.

  5. What is the difference between a heap table and a clustered table?

    • A heap table does not have a clustered index, while a clustered table stores data rows based on the clustered index key.

  6. What is the difference between a temporary table and a table variable?

    • A temporary table is a physical table that exists temporarily in the database, while a table variable is a variable that stores a table-like structure in memory.

  7. What is a query hint in SQL?

    • A query hint is a directive given to the SQL engine to influence how it executes a query, often used to optimize performance.

  8. What is a correlated subquery?

    • A correlated subquery is a subquery that refers to columns from the outer query, making it dependent on the outer query for its execution.

  9. What is a derived table in SQL?

    • A derived table is a temporary table created within a query using the result set of a subquery, often used in complex queries.

  10. What is the difference between a full outer join and a cross join?

    • A full outer join returns all rows from both tables, including those without matches, while a cross join returns the Cartesian product of both tables.


SQL with Data Science and Analytics

  1. How can you perform data analysis using SQL?

    • By using aggregate functions, window functions, and complex queries to summarize and analyze data.

  2. What is the purpose of window functions in SQL?

    • Window functions perform calculations across a set of rows related to the current row, often used for running totals, ranks, and moving averages.

  3. What is the difference between RANK() and ROW_NUMBER() functions?

    • RANK() assigns a rank to each row with potential gaps, while ROW_NUMBER() assigns a unique number to each row without gaps.

  4. How can you calculate a running total in SQL?

    • By using window functions like SUM() with the OVER() clause.

  5. What is the purpose of the LAG() and LEAD() functions in SQL?

    • LAG() returns the value of a previous row, while LEAD() returns the value of a subsequent row, useful for time series analysis.

  6. What is the purpose of the ROLLUP and CUBE operators in SQL?

    • ROLLUP and CUBE are used for generating multi-level summaries of data, often used in reporting and data analysis.

  7. What is a pivot query in SQL?

    • A pivot query transposes rows into columns, often used for summarizing data in a more readable format.

  8. What is the purpose of the GROUP BY clause in SQL?

    • The GROUP BY clause groups rows with the same values into summary rows, often used with aggregate functions.

  9. What is the difference between the GROUP BY and PARTITION BY clauses in SQL?

    • GROUP BY groups rows to aggregate data, while PARTITION BY divides the result set into partitions for window functions.

  10. How can you perform time series analysis using SQL? - By using window functions, LAG(), LEAD(), and date functions to analyze trends and patterns over time.

  11. What is the difference between a scalar subquery and a correlated subquery? - A scalar subquery returns a single value, while a correlated subquery references columns from the outer query and returns values based on each row of the outer query.

  12. What is the purpose of the COALESCE function in SQL? - COALESCE returns the first non-NULL value from a list of expressions, often used for handling NULL values.
